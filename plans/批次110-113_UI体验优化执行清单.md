## 1. 角色管理（role.html / role.js）【批次 111】

### 1.1 系统管理员标签单行显示，不折行
**问题定位**
- 编辑页：`role.js` `openEdit()` 第 243 行，把 `<span class="badge-sys">系统管理员</span>` 写入 `metaBox`（只读信息区）。
- 列表：`role.js` `cardHtml()` 第 93 行，badge 嵌在 `.card-name`（flex 行）内。
- CSS：`role.html` 第 47 行 `.badge-sys` 仅有 `padding` / `border-radius`，**未禁止换行**，窄容器下中文 5 字会折成「系统管 / 理员」。

**修改方案**
- 在 `role.html` 的 `.badge-sys` 增加 `white-space:nowrap; flex-shrink:0;`，同时作用于列表与编辑页（一条规则覆盖两处）。
- 若 `.card-name`（第 46 行，已是 `display:flex; gap:8px`）在极小屏仍折行，可给 badge 加 `flex-shrink:0` 保证不被压缩。

**验收**：编辑页「系统管理员角色」信息区、列表卡片中的「系统管理员」徽标均在一行完整显示，任何屏宽不折行。

### 1.2 权限树「选择框」与「权限名称」轻微错位
**问题定位**
- 结构：`role.js` `buildTreeHtml()` 第 131–138 行生成 `.trow`（flex, `gap:6px`, `align-items:center`）→ `.tlabel-wrap`（flex, `align-items:center`, `gap:8px`）→ `.tcb`（复选框）+ `.tlabel`（名称）。
- CSS：`role.html` 第 111 行 `.tcb{width:18px;height:18px;flex-shrink:0;accent-color:var(--primary);cursor:pointer}`，第 112 行 `.tlabel`。
- 根因：原生复选框在 flex 中虽有 `align-items:center`，但 `accent-color` 渲染的勾选框视觉中心与文字基线不完全重合；且复选框可能存在浏览器默认外边距，造成「略微不对齐」。

**修改方案**
- `.tcb` 增加 `margin:0; align-self:center;`，并与 `.tlabel` 统一基线（可加 `vertical-align:middle`）。
- 确认 `.trow` 与 `.tlabel-wrap` 均为 `align-items:center`（已满足），仅消掉复选框默认外边距与尺寸偏差即可。
- 如仍有亚像素偏差，可将复选框改为固定 `line-height` 与文字一致的 `height`，或统一用 `accent-color` + `width/height` 一致尺寸。

**验收**：每个权限节点的复选框与权限名称在竖直方向严格居中对齐，无上/下偏移。

### 1.3 角色列表「引用人员」点击热区过小
**问题定位**
- `role.js` `cardHtml()` 第 95 行：`引用 <span class="link-like" onclick="openPeopleById('...')">N 人</span>`。
- CSS：`role.html` 第 51 行 `.link-like{color:var(--primary);font-weight:600;cursor:pointer}` 仅包裹「N 人」少量文字，实际可点区域极小，移动端点不准。

**修改方案（已确认：整行点击 = 方案 B）**
- 让整行卡片主体点击即打开该角色的引用人员抽屉：`cardHtml()` 第 90–98 行外层 `.card`/`.card-row` 绑定 `onclick="openPeopleById('...')"`（推荐绑在 `.card-main` 上，避免误触右侧按钮区域）。
- 右侧「编辑 / 删除」按钮保留各自 `onclick`，并在其处理函数内 `e.stopPropagation()`（或在 `onclick` 中加 `event.stopPropagation()`），阻止冒泡触发整行打开引用人员。
- 点击卡片主体打开「引用人员」而非「编辑」，与诉求一致；编辑/删除仍走右侧按钮。

**验收**：角色列表任意卡片主体点击即可打开该角色引用人员抽屉；点击右侧编辑/删除按钮不受影响（不误触整行）。

## 2. 人员管理：选中角色的勾选框蓝色阴影外框（user.html）【批次 110】

**问题定位（根因已确认）**
- `user.html` 第 104–108 行：`.role-chip input{...; box-shadow:none; outline:none !important}`，意图去除复选框阴影。
- 但同文件第 77 行：`.field input:focus, .field select:focus{...; box-shadow:0 0 0 3px rgba(22,119,255,.12)}` —— 特异性 `(0,2,1)` **高于** `.role-chip input` 的 `(0,1,1)`，聚焦/选中时复选框仍被套上蓝色 focus 阴影外框。

**修改方案**
- 在角色勾选框上覆盖 focus 阴影，遵循仓库 RULES（去除**点按**蓝色高亮，但**保留键盘 `:focus-visible`** 可见焦点）：
  ```css
  .role-chip input:focus:not(:focus-visible){ box-shadow:none; }
  ```
  若需稳妥压制，可提升特异性，例如 `.field .role-pick input:focus{ box-shadow:none; }` 或保留 `!important`。
- 同时确认 `.role-chip.on`（第 106 行，选中态蓝边 + 浅蓝底）为预期样式，不被误删。

**验收**：人员新增/编辑页勾选角色时，方形复选框**不再**出现蓝色阴影外框；用键盘 `Tab` 聚焦时仍可见焦点环（无障碍不退化）。

## 3. 权限管理页面（permission.html / permission.js）【批次 112】

**现状行结构**（`permission.js` `buildTreeHtml()` 第 117–118 行）：
```
caret + typeTag + label + badge(已配置/未配置) + sw(启停) + addBtn(＋) + editBtn(编辑) + delBtn(删除)
```

### 3.1 把「启停」与「删除」统一移入编辑
- **行内移除**：删除第 95–96 行的 `sw`（启停开关）与第 98 行的 `delBtn`（删除）拼接。
- **编辑页保留启停**：`permission.html` 第 155–158 行已有「启用」`.switch-row`，保留。
- **编辑页新增删除入口**：在 `permission.html` 的编辑 `.sheet` 底部新增「删除节点」按钮（danger 文本/幽灵按钮），由 `permission.js` `openEdit()` 控制显隐与可用状态。

### 3.2 已配置的项不可删除
- 「已配置」= `REG.isCodeConfigured(code)` 为 `true`（已登记权限注册表）。
- 编辑页删除按钮：当 `isCodeConfigured(code)` 为真 → **禁用/隐藏**删除按钮，并提示「已配置节点不可删除」；同时保留「有子节点不可删」的现有校验。
- 删除确认 `openConfirm()`（`permission.js` 第 277 行）现有逻辑已禁止有子节点删除，新增 `已配置 → 不可删除` 分支（无论由编辑内按钮还是其它入口触发，最终 `doDelete()` 前都再校验一次）。

### 3.3 名称 / code 展示完全 + 按钮间距缩小
- **不截断**：当前 `.tlabel`（第 51 行）`overflow:hidden;text-overflow:ellipsis;white-space:nowrap`、`.tsub`（第 52 行）同样截断。改为：
  ```css
  .tlabel{ white-space:normal; word-break:break-word; overflow:visible; }
  .tsub{ white-space:normal; word-break:break-word; overflow:visible; }
  ```
  保留 `title` 属性供 hover 查看全文。
- **始终显示 code**：目前中文模式只显名称、code 仅在英文模式作副标题。建议行内**主显名称 + 副显 code**（如 `op_company_export`），两者都不截断，把空间让给名称/code。即 `buildTreeHtml()` 中无论中英文都设置 `subText = code`。
- **缩小按钮间距**：移除行内 `sw` + `delBtn` 后，行剩余 `caret + typeTag + label + badge + addBtn(＋新增子节点) + editBtn`（**已确认保留「＋新增子节点」按钮**，仅移走启停与删除），空间已大幅释放；再把 `.trow` 的 `gap`（第 40 行，现 8px）缩到 4–6px，`.icon-btn`（第 66 行）可按需微调。

**验收**：权限树每行名称与 code 完整可见（不省略号）；行内仅保留「展开/类型/名称-code/已配置徽标/＋/编辑」；启停与删除统一在编辑弹层内；已配置节点在编辑内删除按钮置灰并提示不可删。

## 4. 基础数据下所有新增/编辑页全屏化【批次 113】

**现状**：`company / department / position / user / project / project-version`（及 `role / permission`）的新增/编辑均为**底部抽屉** `.sheet`（`department.html` 第 74–102 行 CSS、第 148–182 行 HTML 为典型样例）。
**目标**：改为**全屏页面**，左上角返回箭头（`goBack()`），风格同 `basic-data.html` / 样板 `profile-edit.html`（顶部 `nav-bar` + 撑满 `100dvh` 的 `.card`）。

### 4.1 通用改造模式（每个受影响页套用）
1. **CSS：`.sheet` 由底部抽屉改为全屏覆盖层**
   ```css
   .sheet{
     position:fixed; inset:0; left:0; right:0; top:0; bottom:0;
     max-width:none; width:100%; height:100dvh;
     border-radius:0; transform:none; transition:none;
     display:flex; flex-direction:column; padding:0; overflow:hidden;
   }
   ```
   （可保留淡入动画，去掉 `translateY` 上滑。）
2. **HTML：`.sheet` 内部加 `nav-bar` + 滚动体 + 底部操作条（单滚动原则）**
   - 顶部：`<div class="nav-bar"><button class="nav-back" onclick="goBack()">‹</button><span class="nav-title" id="sheetTitle">新增</span></div>`（标题仍写 `#sheetTitle`，位置移入 nav-bar）。
   - 中部：原表单包一层 `.sheet-body{ flex:1; overflow:auto; padding:16px 14px; }` —— 这是**全屏页唯一的滚动容器**。
   - 底部：`.sheet-actions` 改为 sticky 底部条（`position:sticky; bottom:0; background:var(--surface); padding:12px 14px calc(env(safe-area-inset-bottom)+12px); border-top:1px solid var(--border)`）。
   - **禁止双滚动（已确认）**：手机屏小，不允许页面滚动内再嵌套滚动。角色/权限页全屏化时，必须去掉权限树自身的内部滚动——`role.html` 第 102 行 `.perm-tree{ ... max-height:42vh; overflow:auto }` 改为去掉 `max-height` 与 `overflow:auto`，让权限树随页面自然展开、由 `.sheet-body` 统一滚动。引用人员 `people-sheet` 仍是**独立覆盖弹层**（非嵌套滚动），可保留。
3. **JS**：`openSheet()/closeSheet()/openAdd()/openEdit()` 仅切换 `.show` 类，逻辑基本不变；标题写入 `#sheetTitle` 照旧。
4. **返回行为**：复用 `goBack()`（auth.js 提供，维护返回栈、返回上一页），符合仓库 RULES 返回按钮规范；**不**硬编码 `location.href`。

### 4.2 受影响页面清单

|---|---|---|
| 公司管理 | `company.html` | 抽屉 → 全屏 |
| 部门管理 | `department.html` | 抽屉 → 全屏（典型样例） |
| 职位管理 | `position.html` | 抽屉 → 全屏 |
| 人员管理 | `user.html` | 抽屉 → 全屏；叠加需求 2 |

**验收**：上述 8 个页面的新增/编辑均为整屏显示，左上角箭头可返回上一页；表单内容超长时由全屏页统一滚动（无嵌套滚动），底部「取消/保存」常驻可见。
