# 卡片时间单行化 + 状态操作名标签 —— 批次 73 起

## 背景与目标

基于批次 69–72 的灰色「状态时间」，本轮做两处精修（已与用户确认口径）：

1. **合并时间行（已确认）**：原首页「创建时间」（批次 24 灰行）+「状态时间」（69–72 灰行）**合并为单行**灰时间。
   - 未处理/未开始 → 显示「创建时间」。
   - 其余状态 → 显示**具体操作名 + 时间**（如「开始处理时间」「完成时间」「转交时间」「上线时间」「结束时间」「取消时间」）。
2. **标签语义化（已确认）**：不再统称「状态时间」，而取该状态对应操作的中文名（来自 `TODO_OPERATION` 字典）+「时间」。
3. **会议特判（已确认）**：`typeCode === 'MEETING'` 时，单行时间标签固定为「**会议开始时间**」（值为 `meetingTime`；若 `meetingTime` 缺失则回落「创建时间 / createdAt」），**不**套用操作名规则（即会议中不显示「开始处理时间」）。
4. **彩色 meta 时间标签保留（默认决策）**：现有彩色 meta（事项 `时间：startTime~completeTime`、会议 `时间：meetingTime`、缺陷 `反馈`）**保留**，仅把灰色时间行合并/改标签。若后续要求彻底只留灰行，再额外移除 meta 时间标签。

## 当前数据模型（分析结论）

- 生命周期流水 `todoLifecycles` 每条含 `operationCode` / `statusCode` / `operateTime`。
- 操作码 → 状态（来自 `app.js` `TODO_ACTION_HANDLERS`）：
  - `start` → `TD_DOING`/`BUG_DOING`/`MT_IN_PROGRESS` ← `TODO_START`(开始处理)
  - `complete` → `TD_DONE`/`BUG_DONE` ← `TODO_COMPLETE`(完成)
  - `handoff` → `BUG_WAIT_DEV` ← `TODO_HANDOFF`(转交)
  - `online` → `BUG_ONLINE` ← `TODO_ONLINE`(上线)
  - `end` → `MT_ENDED` ← `TODO_END`(结束)
  - `cancel` → `MT_CANCELLED` ← `TODO_CANCEL`(取消)
  - `reset` → `TD_TODO`/`BUG_TODO`/`MT_NOT_STARTED` ← `TODO_RESET`(重置)
  - `create`/`edit` → 写流水；`edit` 为 `TODO_EDIT` 且 `statusCode` 不变
- `TODO_OPERATION` 字典（`dictionary.js`）：`TODO_CREATE`创建 / `TODO_EDIT`编辑 / `TODO_START`开始处理 / `TODO_COMPLETE`完成 / `TODO_HANDOFF`转交 / `TODO_ONLINE`上线 / `TODO_DELETE`删除 / `TODO_CANCEL`取消 / `TODO_END`结束 / `TODO_RESET`重置。
- 初始状态集合：`TD_TODO`、`BUG_TODO`、`MT_NOT_STARTED`（未处理/未开始）。
- 现状（需改造）：
  - `todo-lifecycles.js` 仅有 `statusOpTimeOf(todo, lifecycles)` 返回**单个时间戳**；调用点在 app.js `renderTodoList`(行1440) 与 report-common.js `loadReportData`(行290)，分别写入 `t.statusOpTime`。
  - app.js `buildTodoCard`(行1503/1505/1514-1515) 渲染**两行**灰时间（创建时间 + 状态时间）。
  - report-common.js `buildTodoCardHtml`(行342/350) 渲染**一行**「状态时间：」。

## 口径定义（单行灰时间）

`getStatusOpLine(todo, lifecycles)` 返回 `{ time, opCode }` 或 `null`，规则：

- **会议特判**（`todo.typeCode === 'MEETING'`）：
  - `todo.meetingTime` 存在 → `{ time: meetingTime, opCode: 'TODO_START' }`（渲染层特判为「会议开始时间」）
  - 否则 `todo.createdAt` 存在 → `{ time: createdAt, opCode: 'TODO_CREATE' }`
  - 否则 `null`
- **初始状态**（`TD_TODO`/`BUG_TODO`/`MT_NOT_STARTED`）→ `{ time: createdAt || CREATE流水.operateTime, opCode: 'TODO_CREATE' }`
- **非初始状态** → 取 `statusCode === 当前` 且 `operationCode !== 'TODO_EDIT'` 的**最新**记录 → `{ time: operateTime, opCode: operationCode }`
- **兜底**（无对应流水）→ `{ time: createdAt, opCode: 'TODO_CREATE' }` 或 `null`

渲染标签：`OP_NAME[opCode] + '时间'`（例：开始处理时间 / 完成时间 / 转交时间 / 上线时间 / 结束时间 / 取消时间 / 创建时间）；**`typeCode==='MEETING' 且 opCode!=='TODO_CREATE'` 时强制标签为「会议开始时间」**。

| 类型 | 状态 | opCode | 标签 | 值 |
|---|---|---|---|---|
| 事项/缺陷 | 未处理/未开始 | TODO_CREATE | 创建时间 | createdAt |
| 事项/缺陷 | 处理中/已完成/待开发/已上线 | TODO_START/COMPLETE/HANDOFF/ONLINE | 开始处理时间/完成时间/转交时间/上线时间 | operateTime |
| 会议 | 任意（有 meetingTime） | TODO_START(哨兵) | **会议开始时间** | meetingTime |
| 会议 | 任意（无 meetingTime） | TODO_CREATE | 创建时间 | createdAt |
| 会议 | 已结束/已取消 | TODO_END/CANCEL | **会议开始时间**（特判覆盖） | meetingTime |
