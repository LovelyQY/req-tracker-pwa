### 批次 105 — 人员管理角色选择 UI 修复

**文件**: `user.html`

**问题1: 角色选框蓝色阴影**
- 根因: `.role-chip input[type="checkbox"]` 在点击时浏览器默认添加 focus 环（`box-shadow` / `outline`），全局 `outline:none` 仅覆盖 outline 属性
- 修复: 在 `.role-chip input` CSS 中追加 `box-shadow: none` 和 `outline: none !important`

**问题2: 角色选择框与备注间距**
- 根因: `.field-hint { margin: -4px 0 10px }` 的负上边距使备注向上侵入 4px
- 修复: 为 `.role-pick` 追加 `margin-bottom: 6px`，或调整 `.field-hint` 在 role-pick 场景的间距

### 批次 106 — 权限树中英双语模式 + 语言配置基础设施

**文件**: `config.js`、`permission.html`、`permission.js`

**架构决策 — 为全站 i18n 预留**

语言偏好不存数据库也不仅存 localStorage，而是以 `config.js` 的 `RT_CONFIG` 为中心、localStorage 为持久层的双层架构：

| 层 | 作用 | 实现 |
|----|------|------|
| 内存 | 全局单一事实来源 | `RT_CONFIG.ui.lang` (默认 `'zh'`) |
| 持久 | 刷新/SW更新后恢复 | `localStorage.setItem('rt_lang', lang)` |
| 未来 | 跨设备同步 / 备份 | IndexedDB 持久层（API 不变，消费者无感） |

**API 设计（config.js）:**
```js
RT_CONFIG.getLang()          // → 'zh' | 'en'
RT_CONFIG.setLang(lang)      // 更新内存 + localStorage + 触发 langchange 事件
RT_CONFIG.langLabel(lang)    // → '中' | 'EN'（供按钮文字）
```
- 页面加载时 `getLang()` 从 localStorage 恢复 → 写入 `RT_CONFIG.ui.lang`
- `setLang()` 同步更新两处，后续全站 i18n 只需替换持久层
- 权限树通过 `RT_CONFIG.getLang()` 读取，无需直接访问 localStorage

**问题3+4（合并）: 权限树支持中/英两版**

需求:
- **中文版**（默认）：树只显示中文名称，type-tag 显示模块/页面/操作，不显示 menuCode
- **英文版**：全显示英文 + 编号（menuCode），type-tag 显示 module/page/op

实现:
1. `config.js`：新增 `ui.lang` + `getLang()`/`setLang()`/`langLabel()` 辅助函数
2. `permission.html`：页面顶部追加语言切换按钮，调用 `RT_CONFIG.setLang()`
3. `permission.js` `buildTreeHtml` 根据 `RT_CONFIG.getLang()` 切换渲染：
   - 中文：`typeLabelMap = { module:'模块', page:'页面', op:'操作' }`，displayName 取 `(menuName || 注册表名 || menuCode).trim()`，隐藏 `.tcode`
   - 英文：类型标签用 `n.nodeType`，主名称 `menuCode`，副标题 `menuName`
4. sheet 表单 select option 同步切换中/英文
5. 按钮文字更新后触发树重渲染

### 批次 107 — 权限新增支持选择已有树节点

**文件**: `permission.html`、`permission.js`

**问题5: 无法选择已有权限树位置**
- 当前 `openAdd()` 无参数，始终创建无父节点的新 page 节点
- 根节点列表中每个行仅有编辑/删除按钮，无「新增子节点」入口

修复:
1. 改造 `openAdd(type, parentCode)` 接受参数：type=默认节点类型, parentCode=预设父节点
2. 每行树节点追加 `+` 按钮（op 类型不显示，因为 op 无子节点）
3. 点击行级 `+` → `openAdd('page'/'op', nodeMenuCode)`，自动选中父节点
4. 顶部「新增」按钮保持原有行为（无父节点）
