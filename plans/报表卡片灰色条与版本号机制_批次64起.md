## 〇、版本号引用机制澄清（先回答用户疑问）

你记得的规则在 RULES.md 第 11 行：

> 统一使用发版脚本，它会一次性完成：7 处版本号同步、生成 CHANGELOG.md、自动提交。

结论先说：**版本字面不是「单个源文件，改一处全站生效」，而是「7 个规范位置」，由 **release.sh 一个命令统一同步。 你记忆中的「规则」指的是这句，但机制本身是「一处命令」而非「一处字面」。

### 版本真值（literal）的 7 处规范位置


| # | 位置 | 字面/变量 |
| --- | --- | --- |
| 1 | index.html | SW_VERSION = 'x.y.z' |
| 2 | index.html | APP_VERSION = 'x.y.z' |
| 3 | sw.js | CACHE = 'req-tracker-vx.y.z' |
| 4 | version.json | "version": "x.y.z" |
| 5 | index.html: app.js?v=x.y.z | 缓存破坏引用 |
| 6 | index.html: styles.css?v=x.y.z | 缓存破坏引用 |
| 7 | index.html: auth.js?v=x.y.z | 缓存破坏引用 |

> RULES.md 第 114 行明确：不要手写 SW_VERSION / APP_VERSION / CACHE 等字面，一律交给 release.sh。所以「改版本」= 跑 ./release.sh，而不是手改某文件。

### 各页面 ?v= 是另一套机制（缓存破坏，约 150 处）

每个 HTML 页面通过 xxx.js?v=x.y.z、styles.css?v=x.y.z 引用静态资源，URL 带版本号是为了**让 Service Worker / 浏览器在发版后拉取新文件**（否则旧缓存导致「刷新仍是旧版」）。

这 **不是** 版本号真值源，只是「随版本变号的引用」。

release.sh 用 AUTH_PAGES / CONFIG_PAGES / BASIC_* / REPORT_* 等循环批量维护这些 ?v=，**但存在盲区**（见下方「批次 65 根因」）。

about.html 第 130 行 var APP_VERSION = '' 是**运行时**从 version.json fetch 赋值的局部变量（第 264 行），它**读取并展示**版本，不是字面真值源。

### 一句话回答

> 版本真值在 index.html / sw.js / version.json 三处（共 7 个规范位置），由 release.sh 一处命令同步；各页面 ?v= 是缓存破坏引用，release.sh 也维护，但有盲区——目前全站 150 处 ?v= 已手工校到 1.3.44，但盲区会在下次发版再次落后，需批次 65 加固。

## 一、批次 64 根因分析 — 报表卡片灰色条

三类报表清单卡片（事项 / 缺陷 / 会议）共用 report-common.js 的 buildTodoCardHtml(t)，渲染类型色条：

```
var typeColorVal = typeColor(typeCode) || '#8c8c8c';   // report-common.js:296
// 卡片根： style="--type-color:' + typeColorVal + '"
```

typeColor 取自 TYPE_CODE_TO_COLOR（report-common.js:27）：

```
function typeColor(code) { return TYPE_CODE_TO_COLOR[code] || '#8c8c8c'; }
```

而 TYPE_CODE_TO_COLOR 只在 loadReportData() 里被 SEED_TYPE.TASK_TYPE** 填充（**report-common.js:259-261）：

```
tasks.push(RT_DICT.getDictByType(RT_DICT.SEED_TYPE.TASK_TYPE).then(function (list) {
  TASK_TYPE_LIST = Array.isArray(list) ? list : [];
  TASK_TYPE_LIST.forEach(function (t) { if (t && t.code) { TYPE_CODE_TO_NAME[t.code] = t.name; if (t.color) TYPE_CODE_TO_COLOR[t.code] = t.color; } });
}).catch(...));
```

问题在这里：

任务统计模块（需求/线上BUG/普通BUG）的类型码 REQ / ONLINE_BUG / COMMON_BUG 属于 SEED_TYPE.TASK_TYPE，**字典带色**（dictionary.js:52-54：#096dd9 / #cf1322 / #ff7a00），所以 report-task.html 的卡片**有颜色**。

三类报表（事项/缺陷/会议）的类型码 TASK_ITEM / BUG / MEETING 属于 SEED_TYPE.TODO_TYPE（dictionary.js:101-103：#fa8c16 / #cf1322 / #1677ff），**根本没被填进 **TYPE_CODE_TO_COLOR，于是 typeColor() 全部回落到灰色 #8c8c8c。

app.js 的待办卡片从正确的 TODO_TYPE 取色 → 待办页有颜色；报表页从 TASK_TYPE 取色 → 灰色。**这就是「与待办页不一致」的根因。**
