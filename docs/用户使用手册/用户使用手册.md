# 用户使用手册（req-tracker-pwa 需求跟踪 PWA）

> 版本：v1.3.48（基线）
> 适用：纯前端、离线优先的需求跟踪 PWA，面向产品/测试/开发团队在移动端与桌面浏览器中记录需求任务与待办、跟踪状态流转、按项目/类型/状态统计。
> 功能依据：`需求说明书.md`；本手册为**最终用户操作指南**，权限系统（RBAC + 数据权限）为待开发功能，相关章节以「⏳ 待开发」标注。

---

## 1. 产品简介

req-tracker-pwa 是一款**无需后端服务器**的渐进式 Web 应用（PWA）：

- **离线可用**：数据存于浏览器本地（IndexedDB），断网也能记录与查看；可「添加到主屏」像 App 一样使用。
- **需求看板**：首页分「需求任务」与「待办」两块，待办再按类型分为任务事项 / 缺陷追踪 / 会议。
- **全生命周期跟踪**：需求任务、待办均支持状态流转与操作流水（时间线）。
- **基础数据**：公司 / 部门 / 职位 / 人员 / 项目 / 项目版本统一管理。
- **统计报表**：按项目 / 类型 / 状态 / 时间统计分布。
- **个人中心 / 备份**：个人信息、账号安全、我的状态；一键导出 / 导入备份防丢。

> ⚠️ 由于无后端，**多设备数据各自独立，不会云端同步**；换设备或防丢请使用「存储与备份」导出文件。

## 2. 安装与首次使用

### 2.1 访问与安装为 PWA

1. 通过 HTTPS 或 localhost 打开应用（GitHub Pages 部署地址或本地服务）。
2. 浏览器地址栏点「安装 / 添加到主屏」图标（Chrome/Edge 右上角、Safari「添加到主屏幕」）。
3. 安装后从主屏图标启动，体验接近原生 App。

### 2.2 首次登录

- 应用首次运行会自动创建默认管理员账号：**账号 `admin`，密码 `123`**。
- 用 `admin / 123` 登录即可开始使用。
- 密码以 SHA-256 加密存储，不以明文保存。

### 2.3 离线说明

- 已访问过的页面与数据在离线时仍可用。
- 新版本发布后，首次切换会拉取最新页面；若页面效果未更新，浏览器硬刷新（`Ctrl/Cmd + Shift + R`）一次即可。

## 3. 登录与会话

登录页（`login/classic.html`）支持两种身份标识：

| 项目 | 说明 |
|---|---|
| 账号 | 人员「账号」字段（新建人员时自动取工号） |
| 工号 | 人员「工号」字段，与账号等价 |

**登录选项**：

- **记住登录**：勾选后会话保存在本地（关闭浏览器仍保留），受下方「登录时长」约束；不勾选则仅当前标签页会话有效。
- **登录时长**：提供 **24 小时（默认）/ 7 天 / 30 天**，决定会话过期时间。
- **退出登录**：在侧边栏 / 个人中心点退出，立即清除会话。

> 会话过期后再次打开任意页面会自动跳回登录页，需重新登录。

## 4. 首页需求看板

首页（`index.html`）明确分两块区域：**「需求任务」区**与**「待办」区**。

### 4.1 需求任务区

卡片展示：任务名称、状态、优先级、类型、所属项目 / 版本、开发人员、状态流转时间线（灰显单行）。

- **筛选**：按类型 / 状态 / 优先级 / 项目 / 关键字过滤列表。
- **操作按钮**随任务状态动态变化（见 §5 生命周期）。
- 顶部「**新建需求任务**」进入任务表单。

### 4.2 待办区（三类型）

顶部类型切换：**任务事项 / 缺陷追踪 / 会议**，三类卡片字段与按钮不同：

| 类型 | 卡片主要字段 |
|---|---|
| 任务事项 | 开发人、开始 / 完成时间 |
| 缺陷追踪 | 关联任务、反馈人 / 时间 |
| 会议 | 会议时间、地点 |

- 顶部「**新建待办**」按当前选中类型打开对应表单。
- 操作按钮同样按类型与状态动态显示（见 §6）。

## 5. 需求任务管理

> 入口：首页「需求任务」区 → 新建 / 点卡片进入详情编辑。

### 5.1 字段说明

| 字段 | 必填 | 说明 |
|---|---|---|
| 任务名称 | ✅ | 1–100 字 |
| 任务描述 | — | 0–1000 字 |
| 任务类型 | ✅ | 字典：需求 / 线上BUG / 普通BUG |
| 优先级 | ✅ | 字典：高 / 中 / 低 |
| 任务状态 | ✅ | 字典：待开发 / 已提测 / 测试中 / 已测完 / 已上线 |
| 所属项目 | ✅ | 选择已有项目 |
| 项目版本 | — | 须归属所选项目（需求组） |
| 开发人员 | — | 可多选人员 |
| 禅道 ID / 子 ID | — | 外部系统关联 |
| 图片 / 附件 | — | 关联媒体库，真实字节独立存储 |

### 5.2 创建 / 编辑 / 删除

- **创建**：填写必填项保存，写入需求任务表。
- **编辑**：修改后保存，记录更新人与更新时间。
- **删除**：删除任务会**级联清理**其关联图片、附件与状态流转流水，避免残留脏数据。

### 5.3 生命周期（8 个操作）

每次状态操作都会记录一条**不可修改**的流水（用于时间线 / 状态标签）：

| 操作 | 含义 |
|---|---|
| 开发提交 (DEV_SUBMIT) | 开发完成提交提测 |
| 测试开始 (TEST_START) | 测试介入 |
| 暂停 (PAUSE) | 暂停测试 |
| 暂停恢复 (RESUME) | 恢复测试 |
| 测试完成 (TEST_DONE) | 测试通过 |
| 上线 (ONLINE) | 已上线 |
| 重置 (RESET) | 回到初始态重新流转 |
| 删除 (DELETE) | 删除任务（含级联） |

> 卡片上的操作按钮依据当前状态动态显示，仅给出当前允许的操作。

## 6. 待办管理

> 入口：首页「待办」区，按类型切换。

### 6.1 三类型与必填项

| 类型 | 必填 | 说明 |
|---|---|---|
| 任务事项 (TASK_ITEM) | 描述 | 通用任务事项 |
| 缺陷追踪 (BUG) | 描述（可选关联任务） | 缺陷 / Bug |
| 会议 (MEETING) | 会议主题 | 会议安排 |

所有待办均须选择**所属项目**；项目版本选填。

### 6.2 三类按钮集（不同）

| 类型 | 可用操作 |
|---|---|
| 任务事项 | 开始处理 / 完成 / 编辑 / 删除 / 重置 |
| 缺陷追踪 | 开始处理 / 完成 / **转交** / **上线** / 编辑 / 删除 / 重置 |
| 会议 | 开始 / **取消** / **结束** / 编辑 / 删除 / 重置 |

> 注意差异：任务事项无「转交 / 上线 / 取消 / 结束」；缺陷无「取消 / 结束」；会议无「完成 / 转交 / 上线」。

### 6.3 流转流水

每次操作写一条**追加式**流水（开始 / 完成 / 转交 / 上线 / 取消 / 结束 / 重置等），用于卡片时间线与状态标签。删除待办时会级联清理其流水。

## 7. 基础数据管理

> 入口：侧边栏「基础数据」（`basic-data.html`），下钻进入各模块。

| 模块 | 关键规则 |
|---|---|
| 公司管理 | 总公司 / 分公司；分公司须归属总公司；**删除前须先删下级分公司** |
| 部门管理 | 归属公司，支持多级层级（同公司自引用）；**防环**（上级不可指向自己的下级）；**删除前须先删下级部门** |
| 职位管理 | 名称 / 编码 / 职级（职级取自字典） |
| 人员管理 | 工号 / 姓名 / 部门 / 职位 / 人员状态；账号自动取工号，默认密码 `123` |
| 项目管理 | 归属部门；名称 / 描述 / 状态 |
| 项目版本管理 | 归属项目（需求组）；名称 / 描述 / 状态 |
| 字典管理 | **只读**，系统枚举由程序维护，页面不可增删改 |

> 所有列表/表单**不展示 32 位系统 ID**，仅显示可读字段（工号 / 编码 / 名称 / 时间等）。

## 10. 存储与备份

> 入口：侧边栏「存储与备份」（`storage-backup.html`）。

- **导出**：将浏览器内全部业务数据导出为备份文件（建议定期备份，或换设备前导出）。
- **导入**：选择备份文件导入恢复数据（重复导入会正确去重 / 覆盖，不生成脏数据）。
- 图片、附件等媒体一并包含。

> 因无后端，**备份是个人数据的唯一跨设备 / 防丢手段**。

## 12. 权限系统（⏳ 待开发：RBAC + 数据权限）

> 本节描述**计划中的**权限功能，当前 v1.3.48 尚未上线；所有登录用户目前权限对等（仅 `admin` 拥有全部数据）。

上线后将提供：

- **角色管理**：创建 / 编辑角色、勾选权限（仅可勾选「已配置」权限）、启停角色；系统管理员角色不可停用 / 删除；仅无人员引用的角色可删除，并展示引用人员便于审计。
- **权限管理**：以树形（模块 > 页面 > 按钮）展示权限节点，标注「已配置（代码真正守卫）」与「新增未真正配置」；权限节点可启停（**停用优先**：停用后该权限全局不生效，高于角色分配）。
- **人员-角色分配**：在人员表单为人员分配一个或多个启用角色，决定其页面与按钮权限。
- **数据权限（部门隔离）**：普通人员仅能查看**本部门及下级部门**的数据；`admin` / 系统管理员角色豁免，可见全部数据。
- **系统管理员默认角色**：`admin` 账号默认拥有「系统管理员」角色，具备全部权限。

> 诚实边界：由于无后端，数据权限是**前端只读过滤（软隔离）**，不能替代服务端鉴权；对直接读取本地数据库的用户无效。该限制将在产品内明示。

## 13. 常见问题（FAQ）

**Q1：换手机 / 清缓存后数据没了？**
A：本应用数据存于本地浏览器，无云端同步。请用「存储与备份」导出，再在新设备导入。

**Q2：忘记 admin 密码？**
A：默认密码为 `123`；若已修改且忘记，需通过浏览器开发者工具清空本地数据后重新初始化（将丢失本地数据，请先备份）。

**Q3：页面打开还是旧版本？**
A：GitHub Pages 有 1–10 分钟 CDN 滞后；硬刷新（`Ctrl/Cmd + Shift + R`）一次通常即可。以仓库 / `CHANGELOG.md` 中的版本为准。

**Q4：删除公司 / 部门失败？**
A：存在下级（分公司 / 子部门）时须先删除下级，系统会提示。

**Q5：字典项为什么不能改？**
A：字典为系统枚举，由程序维护，页面只读，保证类型 / 状态等一致性。

**Q6：待办的按钮为什么和别人不一样？**
A：任务事项 / 缺陷 / 会议的可用操作本就不同（见 §6.2），且按钮随状态动态显示。

## 14. 术语表

| 术语 | 说明 |
|---|---|
| PWA | 渐进式 Web 应用，可安装到主屏、离线可用 |
| 需求任务 | 首页「任务」区主体，归属项目 / 版本，含完整生命周期 |
| 待办 | 首页「代办」区主体，按类型分任务事项 / 缺陷 / 会议 |
| 字典 | 系统枚举（任务类型 / 优先级 / 状态等），仅程序维护、页面只读 |
| 32 位 ID | 所有数据记录的内部主键（十六进制串），对用户不可见 |
| 级联删除 | 删除任务 / 待办时一并清理其图片、附件与流转流水 |
| ⏳ 权限 / 数据权限 | 待开发：角色 / 按钮 / 部门隔离的访问控制 |
