# 测试用例（req-tracker-pwa 需求跟踪 PWA）

> 版本：v1.3.48（基线）
> 范围：本文覆盖**当前已上线的全部功能**与**待开发的权限系统（RBAC + 数据权限）**的测试用例。
> 需求依据：`需求说明书.md`；设计依据：`设计说明书.md`；数据结构依据：`数据库设计说明书.md`；权限设计依据：`plans/权限功能_PLAN.md`；发版规则依据：`RULES.md`。
> 用例命名全小写 `snake_case` 与权限 `code` 体系保持一致；状态/类型 code 以字典种子为准。

---

## 1. 测试范围与策略

### 1.1 测试范围

| 类别 | 覆盖模块 |
|---|---|
| 功能测试 | 登录与会话、首页需求看板、需求任务管理、待办管理（三类型）、基础数据（公司/部门/职位/人员/项目/项目版本/字典）、统计报表、个人中心、存储与备份、关于与更新日志、权限系统（RBAC + 数据权限，待开发） |
| 非功能测试 | 离线可用/PWA、性能、安全、兼容性、可维护性与发版 |
| 规则测试 | RULES.md 强规则（版本 bump、静态资源 `?v=`、32 位 ID、导航约定、权限强规则） |

### 1.2 测试类型约定

| 类型 | 说明 |
|---|---|
| 功能 | 正常业务流程与交互验证 |
| 边界 | 字段长度/数量/空值/极值 |
| 异常 | 非法输入、外键失效、并发/中断 |
| 安全 | 密码、会话、权限、数据权限过滤 |
| 兼容 | 浏览器/移动端交互约定 |
| 规则 | 发版与编码约束 |

### 1.3 优先级约定

| 优先级 | 含义 |
|---|---|
| P0 | 核心主流程，失败即阻塞发布 |
| P1 | 重要功能/异常分支 |
| P2 | 边界/兼容性/细节 |

### 1.4 统一前置与假设

- 测试环境：现代浏览器（Chrome/Edge/Safari/华为浏览器），HTTPS 或 localhost 安全上下文。
- 首次运行已幂等播种默认管理员 `admin / 123`（`ensureDefaultAdmin`），密码以 SHA-256 存储。
- 字典枚举由代码自动播种（`seedDict`），页面只读。
- 所有表主键为 32 位十六进制 `genId()`，对外隐藏，不展示于任何列表/表单。
- 权限功能上线前，所有登录用户权限对等；上线后按 §4 执行。
